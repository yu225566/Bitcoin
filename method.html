四視角特徵規劃 × 碩士論文架構建議

鏈上特徵 + 總體經濟特徵的完整使用規劃，以及論文貢獻的最佳排序

鏈上特徵（優先 3 個）
mvrv_z_score
識別市場高估/低估週期，與 OHLCV 資訊完全獨立（計算自已實現市值，非價格本身）
log_cm_active_addr_cnt
網路使用者規模代理指標，呼應 Metcalfe 定律，文獻引用頻率最高
hash_rate_mom14
礦工信心的前瞻性動量指標，14 日變化率比水準值更平穩且更有預測力
總體經濟特徵（優先 3 個）
log_VIX
全球風險偏好代理，BTC 與 VIX 呈負相關（避險情緒↑ → BTC↓）
DXY_diff
美元強弱一階差分，BTC 與強勢美元負相關，差分後近平穩
US10Y_diff
無風險利率變化，機構投資者估值基準，2024 ETF 通過後相關性↑
❌ 不建議加入（原因）
• cm_market_cap_usd：= Price × Supply，與 Close 線性相關，完全冗餘
• cm_supply_current：高度單調遞增，無變異，無預測力
• log_cm_hash_rate（水準值）：改用動量版本 hash_rate_mom14，含更多前瞻資訊
鏈上萃取方式
MVRV → Z-Score
原始 MVRV 有長期上升趨勢（非平穩）。Z-Score = (MVRV − 365日均) / 365日標準差，消除趨勢同時保留週期偏離信號
Active Addresses → log
地址數分佈嚴重右偏（熊市幾萬 vs 牛市幾百萬），log1p 轉換讓分佈接近常態，梯度更穩定
Hash Rate → 14日動量
絕對算力意義有限，14日變化率 = (HR_t − HR_{t-14}) / HR_{t-14} 捕捉礦工信心加速/減速，是領先指標
總體萃取方式
VIX → log(VIX)
VIX 右偏，高波動時數值暴漲。log 轉換讓 10→20 與 50→100 的差距對等，模型學習更穩定
DXY → 一階差分
美元指數水準值非平穩，但日差分（∆DXY）通常平穩（ADF 可確認）。差分反映當日美元強弱「變化速度」
US10Y → 一階差分
同上，利率水準非平穩但差分平穩。∆US10Y 反映聯準會政策預期的日頻衝擊效果
平穩性檢定
對所有新特徵跑 ADF + KPSS。mvrv_z_score、hash_rate_mom14、DXY_diff、US10Y_diff 預期平穩；log_VIX、log_active_addr 可能仍非平穩 → 對非平穩者補做 FFD（d* 與 OHLCV 的格搜流程一致）
縮放（Z-score）
只在訓練集計算 mean 和 std，驗證集和測試集套用相同參數。這是防止未來洩漏最容易忽略的步驟。
日期對齊
VIX/DXY/US10Y 週末無資料。BTC 週七天交易。以 BTC 日期為主鍵做 left merge，週末的總體特徵用 ffill（沿用上週五收盤值）填補，不用 interpolate（會用到週一資料）。
缺失值策略
鏈上特徵缺失率通常 < 1%，ffill → bfill 即可。若某特徵缺失 > 5%（如 mvrv 2018 年初），需在論文說明並考慮縮短起始日期或刪除該欄。
訓練/驗證/測試分離
縮放、ffill 參數統一只從訓練集計算。每輪走前驗證的訓練集範圍不同，意味著每輪都要重新計算縮放參數，不能跨輪共用。

聯合效益，是否有交互增益？
所有特徵集均使用相同的 12 種架構，input_shape 的特徵維度隨特徵集改變，其餘超參數（n_ts=7、Dropout=0.2、Adam lr=0.001）固定不動。損失函數消融放在最佳特徵集確定後再進行（見論文架構建議）。
關於損失函數優化的排序：放到最後是正確的
損失函數優化（DirHuber）應該在「最佳特徵集確定後」才進行，原因是：如果你用了次優的特徵集來評估損失函數，結論可能不穩健。正確的邏輯是：先找到最能「看見信號」的特徵，再優化「學習方式」。把 C6（損失函數）放在最後，整個論文的因果邏輯才完整封閉。

import yfinance as yf
import pandas as pd
import numpy as np
from statsmodels.tsa.stattools import adfuller

# ── 1. 下載總體經濟特徵 ──────────────────────────────
macro_tickers = {"VIX":"^VIX", "DXY":"DX-Y.NYB", "US10Y":"^TNX"}
macro_dfs = []
for name, ticker in macro_tickers.items():
    df = yf.download(ticker, start="2018-02-01", end="2025-12-31",
                     auto_adjust=True)["Close"].rename(name)
    macro_dfs.append(df)
macro = pd.concat(macro_dfs, axis=1)
macro.index = pd.to_datetime(macro.index).date

# ── 2. 對齊 BTC 日頻（填補週末）─────────────────────
btc_dates = pd.DataFrame(index=pd.date_range("2018-02-01","2025-12-31").date)
macro = btc_dates.merge(macro, left_index=True, right_index=True, how="left")
macro = macro.ffill()  # 週末沿用週五收盤值（不用 bfill 避免前視）

# ── 3. 特徵萃取 ───────────────────────────────────
macro["log_VIX"]   = np.log1p(macro["VIX"])
macro["DXY_diff"]  = macro["DXY"].diff()    # 一階差分
macro["US10Y_diff"]= macro["US10Y"].diff()  # 一階差分

# ── 4. 平穩性確認 ─────────────────────────────────
for col in ["log_VIX","DXY_diff","US10Y_diff"]:
    adf_p = adfuller(macro[col].dropna())[1]
    print(f"{col}: ADF p-value = {adf_p:.4f} → {'平穩 ✓' if adf_p<0.05 else '非平穩 → 需再差分'}")

# ── 5. Z-score 縮放（只用訓練集參數）────────────────
TRAIN_END = "2021-12-31"
train_mask = macro.index <= pd.to_datetime(TRAIN_END).date

scale_cols = ["log_VIX","DXY_diff","US10Y_diff",
              "mvrv_z_score","log_cm_active_addr_cnt","hash_rate_mom14"]

scaler_params = {}
for col in scale_cols:
    if col in macro.columns:
        mu  = macro.loc[train_mask, col].mean()
        std = macro.loc[train_mask, col].std()
        macro[f"{col}_z"] = (macro[col] - mu) / std
        scaler_params[col] = (mu, std)  # 存起來供測試集使用

print("前處理完成，特徵維度：", macro.shape)
⚠️ 走前驗證時的縮放注意事項
每一輪走前驗證的訓練集範圍不同（W1 訓練至 2022/12，W2 至 2023/12，W3 至 2024/12），需要對每輪分別重新計算 scaler_params。不可以把 W1 輪的縮放參數沿用到 W2 輪，否則 W2 的測試集縮放用了「未來」的訓練資料統計量，造成洩漏。
